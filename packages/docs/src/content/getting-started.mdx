# Getting Started

Manti UI ships as a small set of packages. For a React app you install two — the
renderer and the stylesheet — and the tokens and behavior machines come along
automatically.

## Install

Install the packages with your package manager of choice:

<InstallTabs packages="@manti-ui/react @manti-ui/styles" />

`@manti-ui/react` pulls in `@manti-ui/tokens` and the behavior machines
(`@manti-ui/folds`) automatically. `@manti-ui/styles` is a **peer dependency** —
you install it alongside the renderer, so you stay in control of the CSS version
and its import order (which matters when you also use Tailwind).

## Import the styles once

`@manti-ui/react` ships **no** CSS side effects, so import the stylesheet
yourself, once, at your app entry. Keeping the import explicit leaves you in
control of order (which matters when you also use Tailwind), and this single line
sets up the cascade layers, design tokens, component styles, and motion tiers.

```ts
// main.tsx / app entry
import '@manti-ui/styles/index.css';
```

## Set the theme

Theming is driven by a `data-theme` attribute on `<html>` (or any ancestor), with
native `light-dark()` resolving the right values. Set it to `light` or `dark`:

```html
<html data-theme="dark">
  …
</html>
```

To avoid a flash, set it before paint with a tiny inline script that reads the
user's stored choice or their system preference — exactly what this site does.

## Render a component

Import any component and render it. Props like `variant` and `size` map to the
public `data-*` anatomy that you can target with your own CSS.

```tsx
import { Button } from '@manti-ui/react';

export function Example() {
  return (
    <Button variant="primary">
      Knead dough
    </Button>
  );
}
```
